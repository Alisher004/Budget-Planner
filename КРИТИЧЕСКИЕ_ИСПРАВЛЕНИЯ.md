## 📊 Технические детали

### Auth Persistence Flow

```
1. User opens app
   ↓
2. Firebase checks localStorage for auth token
   ↓
3. onAuthStateChanged fires
   ↓
4. If token valid → setUser(user), setLoading(false)
   If token invalid → setUser(null), setLoading(false)
   ↓
5. Component checks: if (!loading && !user) → redirect
   ↓
6. If user exists → show dashboard
```

### Burger Menu Z-Index Hierarchy

```
z-[60] - Burger button (absolute, highest)
z-[50] - Sidebar (fixed, middle)
z-[45] - Overlay (fixed, lowest, only when open)
z-[40] - Main content (default)
```

### Position Strategy

```
Burger button: absolute
  - Позиционируется относительно родителя
  - НЕ блокирует весь экран
  - Всегда кликабельна

Sidebar: fixed
  - Фиксирована относительно viewport
  - Правильно для sidebar
  - Slide-in анимация на mobile

Overlay: fixed
  - Покрывает весь экран
  - Только когда menu открыто
  - Закрывает menu при клике
```

## ⚠️ Если Auth НЕ работает

### Проблема: Пользователь разлогинивается после обновления

**Возможные причины:**

1. **Firebase config неправильный**
   ```bash
   # Проверьте .env.local
   NEXT_PUBLIC_FIREBASE_API_KEY=...
   NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=...
   NEXT_PUBLIC_FIREBASE_PROJECT_ID=...
   ```

2. **Cookies заблокированы**
   ```
   Проверьте: DevTools → Application → Cookies
   Должны быть cookies от Firebase
   ```

3. **Incognito mode**
   ```
   В режиме инкогнито persistence может не работать
   Тестируйте в обычном режиме
   ```

4. **Browser cache**
   ```bash
   # Очистите кэш
   DevTools → Application → Clear storage → Clear site data
   ```

### Проблема: Мгновенный редирект на /login

**Решение:**
```typescript
// Проверьте, что loading state используется
if (!loading && !user) {
  router.push('/login');
}

// НЕ должно быть:
if (!user) {  // ❌ Неправильно
  router.push('/login');
}
```
