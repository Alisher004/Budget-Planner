✅ Нет несоответствия между UI и логикой

### ✅ 3. Daily Expenses Firestore - РЕАЛИЗОВАНО
**Проблема:** Ежедневные расходы сохранялись только в localStorage

**Решение:**
- Реализовано сохранение в Firestore collection `expenses`
- Структура документа:
  ```typescript
  {
    userId: string,
    amount: number,
    category: string,
    note: string,
    date: string,
    timestamp: number,
    createdAt: serverTimestamp()
  }
  ```
- Функции: `fetchExpenses()`, `handleAddExpense()`, `handleDeleteExpense()`
- Query с фильтром: `where('userId', '==', user.uid)`
- Сортировка в памяти (не требует индекса)

**Файлы:**
- `app/dashboard/daily/page.tsx`
- `app/dashboard/page.tsx`
- `app/dashboard/analytics/page.tsx`

**Результат:**
✅ Расходы сохраняются в Firestore  
✅ Данные персистентны после обновления  
✅ Каждый пользователь видит только свои расходы  
✅ Loading states и Toast notifications

**Проблема:** Ошибка загрузки расходов из-за отсутствия индекса

**Файлы:**
- `app/dashboard/daily/page.tsx`

**Результат:**
✅ Расходы загружаются без ошибок  
✅ Не требуется индекс в Firestore  
✅ Быстро для < 1000 записей

## 🏗️ Архитектура

### Firestore Collections

#### expenses
```typescript
{
  userId: string,           // Required, indexed
  amount: number,           // Required
  category: string,         // Required
  note: string,             // Optional
  date: string,             // YYYY-MM-DD
  timestamp: number,        // Unix timestamp
  createdAt: Timestamp      // Server timestamp
}
```

#### users
```typescript
{
  email: string,
  isPremium: boolean,
  trialStart: timestamp,
  createdAt: timestamp
}
```

#### payments
```typescript
{
  userId: string,
  userEmail: string,
  status: 'pending' | 'approved' | 'rejected',
  createdAt: timestamp
}
```
